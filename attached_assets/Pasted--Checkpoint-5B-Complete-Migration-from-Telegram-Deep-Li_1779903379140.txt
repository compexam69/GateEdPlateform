🟢 Checkpoint 5B: Complete Migration from Telegram Deep Links to Direct Private Telegram Message/Lecture Links

Goal: Completely replace the existing Telegram deep-link system (Chat ID + Message ID + tg:// protocol) with direct private Telegram message/lecture links while preserving all existing functionality, analytics tracking, permissions, and student lecture workflows without breaking the current system.

Context: The current Telegram lecture integration uses:

- Telegram Chat ID
- Telegram Message ID
- tg:// deep-link generation logic

Some features are already implemented, including:

- Lecture click tracking
- Telegram lecture opening flow
- Analytics logging
- Content Editor integration
- Student-side “Watch Lecture” functionality

However, the current deep-link approach has several issues:

- Inconsistent browser/PWA behavior
- Telegram app dependency
- Complex admin workflow
- Hard-to-maintain link generation logic
- Inconsistent mobile/web fallback support

New Requirement:

- Completely replace Telegram deep-link logic with direct private Telegram message/lecture links.
- Remove dependency on:
  - Chat ID
  - Message ID
  - tg:// protocol links
- Use direct private Telegram message/video links as the primary and only lecture delivery system.
- Ensure only students joined in the private Telegram group/channel can access lectures.
- Preserve all existing analytics, permissions, and workflows without breaking any functionality.
