# 🟢 Checkpoint X: Admin Subjects Global Hierarchy Search & Mobile-First Navigation

## Goal

Implement a **live global search/filter system** on the **Admin Dashboard → Content Editor → Subjects page** that allows Admins and Super Admins to instantly find and navigate to any **Subject, Chapter, or Topic** from a single search bar.

The search must:

* Search across the entire content hierarchy.
* Match Subject names, Chapter names, and Topic names.
* Update results in real time as the user types.
* Display complete hierarchy context:

```text
Subject
└── Chapter
    └── Topic
```

so administrators always know where a result belongs.

The implementation must be:

* Mobile-first
* Responsive
* Fast
* Scalable
* Production-ready

without degrading Content Editor performance.

---

# Context Analysis

## Current Problem

As content grows:

```text
Subjects
  → Chapters
      → Topics
```

become difficult to navigate.

Administrators currently must:

```text
Open Subject
↓
Open Chapter
↓
Locate Topic
```

manually.

This becomes inefficient when managing:

* Large course catalogs
* Hundreds of Chapters
* Thousands of Topics

There is currently no centralized search experience.

## Guidance for You (Supabase SQL Editor)

**Do not run SQL immediately.**

In most implementations, this feature can be built using the hierarchy data already loaded into the Content Editor.

Therefore:

```text
Likely SQL Required: NO
Likely RLS Changes Required: NO
Likely Schema Changes Required: NO
```

Only consider SQL indexes if:

* Search becomes server-side.
* Dataset becomes extremely large.
* Performance testing shows slow query execution.

The implementer must first determine whether the search will be client-side or server-side and document that decision in the final implementation report.
