🟢 Checkpoint 6: Quiz & Question Editor Flexible Test Creation, Bulk Import System & Mobile Responsiveness

Goal: Implement a fully flexible and production-ready Quiz & Question Editor system that allows Admins and Super Admins to create tests with or without video solutions/QR codes while also improving the Bulk Import workflow, question PDF conversion process, and mobile responsiveness for editor action buttons.

Context: Currently:

* The Quiz & Question Editor enforces video solutions or QR code-related fields in some workflows.
* Many tests only require:

  * Questions
  * Options
  * Correct answers
  * Text solutions/explanations
    without video content.

Additionally:

* “Bulk Import” and “Add Questions” buttons are not properly responsive on mobile devices.
* Admins do not have a clear workflow for generating/importing bulk questions from question PDFs.

New Requirement:

* Video solutions and QR codes must become fully optional.
* Admins and Super Admins should be able to:

  * Create tests with or without video solutions
  * Import large sets of questions easily
  * Generate bulk import files from question PDFs
* Student view must automatically adapt depending on whether video solutions exist.
* Mobile responsiveness for editor controls must be fully optimized.

Instructions:

1. Analyze Existing Quiz & Question System:

* Review:

  * Quiz Editor
  * Question Editor
  * Bulk Import workflow
  * Database schema
  * Validation logic
  * Video/QR handling
  * Student-side rendering
  * Existing gating logic
  * Mobile responsive layouts
* Identify:

  * Where video/QR fields are mandatory
  * Which validations enforce them
  * How bulk import currently works
  * Which layouts/buttons break on mobile

2. Make Video Solutions & QR Codes Optional:

* Update the system so:

  * Video URLs become optional/nullable
  * QR generation becomes optional
* Ensure:

  * Tests save successfully without videos
  * Existing video-based tests continue working
  * Validation no longer blocks text-only tests

3. Update Student Solution Rendering:

* Refactor student-side logic so:

  * Text solutions display even without videos
  * QR codes automatically hide when video is absent
  * Empty placeholders are never shown
* Ensure:

  * Clean and readable UI
  * Stable rendering across devices

4. Improve Bulk Import Workflow:

* Ensure Bulk Import supports:

  * Questions
  * Options
  * Correct answers
  * Text explanations
  * Optional video URLs
  * Optional QR/video fields
* Validate:

  * Required fields
  * File structure
  * Duplicate questions
  * Missing data
* Prevent:

  * Corrupted imports
  * Invalid question formats
  * Partial save failures

5. Improve Mobile Responsiveness:

* Refactor:

  * “Bulk Import”
  * “Add Questions”
  * Related action buttons
* Ensure:

  * Fully responsive mobile layout
  * Proper spacing and touch targets
  * No overlap or overflow
* Consider:

  * Vertical stacking
  * Sticky action footer
  * Full-width mobile buttons

6. Preserve Existing Functionality:

* Ensure implementation does NOT break:

  * Lecture tests
  * Topic tests
  * Subject tests
  * Full tests
  * Existing gating/progression logic
  * Analytics/reporting
  * Existing permissions

  * Correct Answer
  * Explanation/Text Solution
  * Video URL (optional)
  * QR/video field (optional)

  * Subject/topic/test mapping fields

  * Proper example rows
  * Correct formatting rules
  * Common mistakes to avoid

  * How to convert question PDFs into bulk import format
  * Which AI/manual tools can help extract questions
  * How to clean extracted text

  * How to structure answers/options correctly
  * How to prepare import-ready files

  * Duplicate questions
