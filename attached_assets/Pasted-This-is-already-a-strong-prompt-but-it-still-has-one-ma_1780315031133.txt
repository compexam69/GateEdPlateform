This is already a strong prompt, but it still has one major weakness:

> It focuses heavily on code auditing, security, cleanup, and stabilization, but it does not force a true production-readiness verification of actual user workflows after fixes are applied.

A developer could technically complete this audit by reviewing code, dependencies, SQL, and security without proving that the application still works correctly from an end-user perspective.

For a production system, you want a **Code Audit + Security Audit + Functional Regression Audit + Production Readiness Validation** combined into one final stabilization checkpoint.

---

# 🟢 Final Stabilization Checkpoint: Full Codebase Audit, Security Hardening, Technical Debt Cleanup & Production Readiness Verification

## Goal

Perform a comprehensive production-grade audit, cleanup, hardening, optimization, and validation of the entire application.

The objective is to identify, document, and safely remediate:

* Dead code
* Bugs
* Logic errors
* Security vulnerabilities
* Permission issues
* RLS weaknesses
* Database inconsistencies
* Performance bottlenecks
* Dependency risks
* Technical debt
* Orphaned modules
* Unused APIs
* Unused database structures

while ensuring:

* No existing functionality breaks
* No regressions are introduced
* The application remains fully operational
* The application is verified as production-ready

## Guidance for You

### Before the Audit

1. Create a full Supabase backup.
2. Export current RLS policies.
3. Export the database schema.
4. Create a staging environment.
5. Ensure test accounts exist for:

   * Super Admin
   * Admin
   * Student

### Supabase SQL Editor

**Do not run SQL preemptively.**

The audit must first determine whether:

* Schema changes are required
* Indexes are required
* RLS changes are required
* Cleanup queries are required

Only run SQL after:

* Findings are documented
* Rollback scripts are prepared
* Migration risks are reviewed

### What You Should Expect at the End

You should receive:

* Complete issue inventory
* Security assessment
* Database assessment
* SQL migration package (if required)
* Rollback scripts (if required)
* Production readiness verdict
* Prioritized remediation roadmap
* Final stabilization checkpoint report
