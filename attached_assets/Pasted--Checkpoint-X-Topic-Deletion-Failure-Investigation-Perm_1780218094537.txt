## Goal

Investigate and permanently resolve the issue where deleting a Topic inside a Chapter fails and displays:

"Unable to delete topic. Try again later."

The Topic remains in the database and continues to appear in the UI.

The objective is to identify the actual root cause of the failure and implement a production-ready fix so that authorized users can permanently delete Topics successfully.

The system must:

* Successfully delete Topics from the database.
* Display accurate success messages only when deletion succeeds.
* Display accurate error messages when deletion fails.
* Maintain database integrity.
* Respect existing permissions and access control rules.
* Prevent silent failures.
* Prevent misleading notifications.
