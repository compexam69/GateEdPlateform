🟢 Checkpoint X: User Management – Move Reinstate User & Add Delete User Action to Three-Dot Menu (Suspended Users Tab)
Goal

Refactor the Suspended Users tab inside:

Admin Dashboard → User Management → Suspended Users

so that:

The existing Reinstate User action is moved into the existing three-dot (⋮) action menu.
A new Delete User action is added to the same three-dot menu.
Delete User permanently removes the selected user from the database after confirmation.
The UI becomes cleaner, more consistent, and aligned with the Active Users action menu pattern.

The implementation must preserve existing permissions, security controls, audit logging, and responsive behavior.

Context Analysis
Current State

Inside the Suspended Users tab:

Reinstate User appears as a standalone action/button.
No Delete User option exists.
Actions are visually inconsistent with the desired overflow-menu pattern.

This creates:

UI clutter
Inconsistent user management workflows
Poor scalability as additional actions are introduced
Desired State

Each suspended user row/card should contain:

Three-Dot Menu (⋮)

Containing:

View User
Reinstate User
Delete User

(or equivalent existing actions)

All user-management actions should be consolidated into the overflow menu.

Important Deletion Requirement

Deletion must be:

Permanent

The selected user should be removed from the system.

The implementation team must first determine:

Scenario A — Hard Delete

User record is permanently removed from database tables.

OR

Scenario B — Soft Delete

User record is marked deleted and hidden from the application.

The implementation report must explicitly state which approach is used.
