🟢 Checkpoint 16.2: Remove Swipe-Down Close Gesture & Restore Native Smooth Mobile Scrolling

Goal: Completely remove the swipe-down-to-close gesture from the Profile Photo Viewer because it continues to interfere with normal mobile scrolling behavior and creates a frustrating user experience.

Context: The swipe-down gesture was originally implemented to mimic native gallery/photo viewer behavior. However, even after multiple optimizations, the feature still causes mobile scrolling issues such as:

* Sticky/unresponsive scrolling
* Delayed touch response
* Scroll lag
* Gesture conflicts
* Accidental scroll blocking
* Poor UX during normal interaction

The current implementation negatively affects the smoothness of the application and irritates users during normal usage.

New Decision:

* Remove the swipe-down-to-close feature entirely.
* Restore fully native-like scrolling and touch responsiveness.
* Keep browser refresh behavior ONLY when the user intentionally performs a long pull-down gesture from the very top of the screen (default browser behavior).
* Prioritize smooth scrolling and responsive touch interactions over gesture-based dismissal.

Instructions:

1. Analyze Existing Gesture & Touch System:

* Review:

  * Swipe-down gesture implementation
  * Touch event listeners
  * preventDefault logic
  * Scroll-lock handling
  * Modal drag animations
  * Pull-to-refresh prevention logic
  * PWA touch handling
* Identify:

  * All gesture-related logic tied to swipe-down close behavior
  * Any event listeners intercepting normal scrolling
  * Any custom drag/gesture state management
  * Any animation logic attached to swipe gestures

2. Completely Remove Swipe-Down Close Functionality:

* Remove:

  * Swipe-down close detection
  * Drag tracking logic
  * Gesture threshold calculations
  * Velocity detection
  * Gesture animation states
  * Swipe-dismiss transitions
  * Any custom touchmove prevention logic
* Ensure:

  * No leftover gesture handlers remain
  * No hidden gesture side effects remain
  * No stale modal state persists

3. Restore Native Mobile Scrolling:

* Ensure:

  * Scrolling works instantly with light/quick touches
  * Users can scroll naturally without delay
  * Touch interactions feel fully native
  * Mobile scrolling is smooth and hardware-accelerated
* Prevent:

  * Sticky scrolling
  * Delayed response
  * Scroll freezing
  * Gesture hijacking

4. Preserve Existing Viewer Features:

* Ensure the following continue working correctly:

  * Tap-to-open photo viewer
  * Close button/manual close
  * Upload/update/delete profile photo
  * Image zooming/scaling (if supported)
  * Existing responsive layouts
  * Existing animations unrelated to swipe-down
* Existing functionality must remain fully stable.

5. Restore Default Pull-to-Refresh Behavior:

* Ensure:

  * Browser refresh only occurs naturally when:

    * User intentionally pulls down from the very top of the page
    * Browser-native pull-to-refresh is triggered
* Prevent:

  * Accidental refreshes during normal scrolling
  * Custom refresh conflicts
  * Partial gesture interception

6. Improve Overall Mobile UX:

* Optimize:

  * Touch responsiveness
  * Modal performance
  * Scroll smoothness
  * Animation performance
* Ensure:

  * No jittering occurs
  * No lag occurs on low-end devices
  * User interactions feel lightweight and fluid

7. Clean Up Gesture-Related Code:

* Remove:

  * Unused gesture utilities
  * Unused animation logic
  * Dead code
  * Unnecessary touch listeners
  * Redundant state management
* Ensure:

  * Cleaner maintainable codebase
  * Improved performance
  * Reduced memory/event overhead

8. Handle Edge Cases:

* Verify stable behavior during:

  * Rapid scrolling
  * Fast tapping
  * Orientation changes
  * PWA mode
  * Low-performance devices
  * Long content/images
* Prevent:

  * Frozen modal states
  * Broken scrolling
  * Background scroll issues

9. Testing & Validation:

* Test:

  * Android browsers
  * iOS browsers
  * Installed PWA mode
  * Portrait/landscape orientation
  * Fast scrolling
  * Rapid interactions
  * Low-end devices
* Verify:

  * Scrolling feels completely smooth and natural
  * No swipe-down gesture remains
  * No accidental refresh occurs
  * Default browser refresh works only from top-of-screen pull-down
  * Existing photo viewer features remain stable
  * No console/runtime errors occur

10. Backend / Storage / SQL Changes:

* Explicitly state that no backend/storage/SQL changes are required unless unexpectedly needed.
* If any related changes are required:

  * Clearly explain them separately.

11. Checkpoint Requirement:

* After completing and verifying this UX stabilization:

  * Create a proper checkpoint/commit immediately.
  * Ensure the mobile scrolling experience is fully stable and production-ready.
  * Provide a final implementation summary including:

    * Files modified
    * Gesture removal changes
    * Scroll responsiveness improvements
    * Performance optimizations
    * Cleanup/refactoring performed
    * Remaining limitations/issues (if any)
