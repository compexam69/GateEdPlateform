This requirement is actually a **responsive layout separation task**, not a simple dropdown change.

After analyzing the requirement, the root issue is:

### Current Situation

✅ Mobile Profile page is well-balanced

✅ Dropdown/collapsible sections help save space on small screens

❌ Desktop Profile page looks underutilized

❌ Large amounts of empty space exist

❌ Users must unnecessarily open dropdowns despite having ample screen space

❌ Desktop layout feels compressed instead of taking advantage of larger viewports

The correct solution is **adaptive rendering by breakpoint**, where desktop and mobile intentionally behave differently.

On mobile devices:
