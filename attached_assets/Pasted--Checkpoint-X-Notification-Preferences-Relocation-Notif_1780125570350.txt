🟢 Checkpoint X: Notification Preferences Relocation & Notification Settings Architecture Refactor

Goal:

Refactor the Notification Settings architecture by moving all Notification Preference controls from the Profile module into the Notifications module, making them accessible exclusively through the ⚙️ (Gear/Settings) button within the Notifications section while preserving all existing notification functionality, user preferences, permissions, and delivery behavior.

Context:

Currently, Notification Preferences are located inside the Profile module.

This creates several UX and architectural issues:

* Notification-related settings are separated from the actual Notifications module.
* Users must navigate to Profile Settings to manage notification preferences.
* Notification management is fragmented across different areas of the application.
* The Notification section contains a Settings (⚙️) button but does not act as the central location for notification configuration.

The platform requires a more intuitive and centralized notification management experience.

New Requirement:

Move all Notification Preference settings currently located in the Profile module into the Notifications section.

Users should manage notification settings through:

Notifications
→ ⚙️ Settings Button
→ Notification Preferences

Important Requirements:

* Remove Notification Preferences from the Profile module.
* Notification Preferences must only be managed through the Notifications section.
* Existing preference values must remain intact.
* No user settings should be lost during migration.
* Existing notification delivery logic must remain unchanged.
