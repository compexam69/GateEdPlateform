🟢 Checkpoint 15: User Management Action Buttons UI/UX Refactor (Mobile-First Responsive Layout Optimization)

Goal: Redesign and optimize the action button layout inside the Admin Dashboard → User Management module to create a cleaner, more organized, modern, and mobile-friendly interface. Fix the current cluttered and misaligned button structure across Pending, Active, and Suspended user sections while maintaining full functionality.

Context:
The current User Management module contains multiple action buttons displayed below each user card:

* View
* Edit
* Approve/Reject
* Suspend/Reinstate
* Role Selection
* Reset

Current issues observed from the existing implementation and attached mobile screenshot:

* Buttons are overcrowded and visually cluttered
* Spacing and alignment are inconsistent
* Mobile layout feels cramped and difficult to use
* Some buttons wrap awkwardly
* Role dropdown and Reset button alignment feels disconnected
* Action hierarchy is unclear
* Large number of visible buttons creates cognitive overload
* Create User and Import Users buttons are not properly optimized for smaller screens
* User cards become visually heavy on mobile devices
* Interface does not feel modern/native-app-like

Since most users/admins may access the system from mobile devices, the User Management module must become mobile-first, highly responsive, scalable, and easy to interact with.

New Requirement:

* Completely redesign the action button structure and layout.
* Create a cleaner and more intuitive UI/UX.
* Optimize for:

  * Mobile usability
  * Better spacing
  * Easier touch interaction
  * Reduced visual clutter
  * Better grouping of actions
  * Modern responsive design
* Preserve all existing functionalities.

* Review:

  * User Management component
  * User card layout
  * Action button rendering logic
  * Role selection dropdown
  * Responsive breakpoints

  * Admin action workflows
* Identify:
