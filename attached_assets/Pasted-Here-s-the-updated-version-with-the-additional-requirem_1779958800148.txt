🟢 Checkpoint 12: Fix Unnecessary Profile Image Reloading During Module Navigation (Including Sidebar Avatar)

Goal: Identify and fix the issue causing the profile image to reload every time the user navigates between modules/pages, including the profile photo displayed in the sidebar/navigation area. Context:

* The profile image now remains visible after refresh/navigation.

Current issue:

* Re-fetching profile data on every route change
* Context/provider re-rendering
* Sidebar/layout remounting

* Re-generating image URLs repeatedly
* Signed URL regeneration
* Cache invalidation problems
* State reset during navigation
* Improper dependency arrays in hooks
* Duplicate API/storage fetch calls
* PWA caching inconsistencies
