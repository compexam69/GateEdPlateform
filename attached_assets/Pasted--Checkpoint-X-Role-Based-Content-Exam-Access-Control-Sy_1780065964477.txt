🟢 Checkpoint X: Role-Based Content & Exam Access Control System (Multi-Role Visibility Permissions)

Goal:

Implement a secure and production-ready role-based access control system for Content and Exam visibility that supports flexible multi-role access selection while preserving strict Super Admin isolation rules and existing permission architecture.

Context:

Currently, the platform does not provide a fully flexible visibility/access control mechanism for content and exams.

There is a new requirement to allow creators to control who can access/view specific:

* Content
* Exams/Tests
* Resources/Lectures (if applicable)

using role-based multi-selection access rules.

Important business rule:

* A Super Admin can view all content created by:

  * Admins
  * Students/users
* BUT:

  * One Super Admin must NOT automatically access another Super Admin’s content.
  * Access between Super Admins must require explicit permission/granted access.

The system must remain secure, scalable, and production-ready while preserving existing content permissions, routing, rendering logic, analytics, and dashboard workflows.

New Requirement:

Add a role-based visibility/access configuration system for content and exams supporting:

* Creator Only
* Admins
* Students
* Super Admins (restricted by explicit permission logic)

The creator should be able to select multiple allowed roles for each content/exam item.

Examples:

* Creator only
* Creator + Admins
* Creator + Students
* Admins + Students
* Everyone except other Super Admins
* Explicitly shared between Super Admins

Important Permission Rules:

1. Creator Visibility

* Content creators must always retain access to their own content.

2. Admin Access

* Admins can only access content/exams allowed for Admin role.

3. Student Access

* Students can only access content/exams allowed for Student role.

4. Super Admin Isolation Rule

* Super Admins can access:

  * Their own content
  * Admin-created content
  * Student-created content
* BUT:

  * One Super Admin CANNOT automatically access another Super Admin’s content.
* Cross-Super-Admin visibility requires:

  * Explicit permission/share access.
