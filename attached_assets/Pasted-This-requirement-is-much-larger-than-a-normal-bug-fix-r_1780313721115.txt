This requirement is much larger than a normal bug-fix request. It is essentially a **full production-readiness audit, QA validation, security review, permission audit, database audit, responsiveness audit, and deployment-readiness assessment** of the entire application.

A good implementation prompt must prevent a superficial review where only a few pages are checked. It should force a systematic audit, evidence-based testing, issue classification, root-cause analysis, and a final go/no-go production decision.

---

# 🟢 Final Pre-Production Audit: Full Application Verification & Production Readiness Assessment

## Goal

Perform a complete end-to-end audit of the entire application to determine whether it is truly production-ready.

The audit must verify:

* Functionality
* Stability
* Security
* Permissions
* Database integrity
* API reliability
* Mobile responsiveness
* Desktop responsiveness
* PWA behavior
* Performance
* Error handling
* UX consistency

The objective is to identify every known issue, regression, broken workflow, hidden bug, permission problem, database issue, responsiveness problem, security risk, and deployment blocker before production release.

This is not a feature implementation task.

This is a comprehensive verification and quality assurance checkpoint.

# Guidance for You (Project Owner)

### Before Running This Audit

Prepare:

* Super Admin test account
* Admin test account
* Student test account
* Sample content
* Sample quizzes
* Sample PDFs
* Sample notifications

This allows complete testing coverage.

### Supabase SQL Editor

**Do NOT run any SQL before the audit.**

First determine:

```text
Are SQL changes actually required?
```

Only after findings are identified should any SQL migration scripts be proposed and reviewed.

### Expected Outcome

At the end of this process you should receive:

1. A complete defect report.
2. A production readiness score/status.
3. Any required SQL scripts.
4. A prioritized bug-fix roadmap.
5. A final go/no-go recommendation for deployment.
