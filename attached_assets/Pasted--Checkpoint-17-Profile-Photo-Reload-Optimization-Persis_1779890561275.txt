🟢 Checkpoint 17: Profile Photo Reload Optimization & Persistent Image Caching

Goal: Fix the issue where the profile photo reloads every time the user opens the Profile module from another module, causing unnecessary loading delays, flickering, repeated network requests, and poor user experience.

Context: The project is functional, but there is a UX/performance issue related to the Profile module:

* Whenever the user navigates to the Profile module:

  * The profile photo reloads again
  * Loading/flickering becomes visible
  * The image fetch/network request happens repeatedly
* This happens even when:

  * The image has already been loaded previously
  * The user has not updated their profile photo
  * The session is still active

This creates:

* Unnecessary network usage
* Slower navigation experience
* Visual flickering
* Poor perceived app performance

The issue is likely caused by:

* Re-fetching image URLs on every navigation
* Missing image caching/state persistence
* Component remount behavior
* Unoptimized state management
* Cache-busting query regeneration
* Re-render loops
* Storage signed URL regeneration on every page load

New Requirement:

* Profile photos should load instantly after the first successful load.
* The image should remain cached/persisted during normal app navigation.
* Re-fetching should only occur when:

  * The user uploads/changes/deletes the profile photo
  * Session/user data changes
  * Cache expires intentionally
* Navigation between modules should feel smooth and instant.

Instructions:

1. Analyze Existing Profile Photo Flow:

* Review:

  * Profile module/component
  * Image loading logic
  * Storage URL generation
  * Signed URL generation
  * State management
  * Cache handling
  * Navigation lifecycle
  * Component mount/unmount behavior
  * Global auth/user store
* Identify:

  * Why the image reloads on every navigation
  * Whether signed URLs regenerate repeatedly
  * Whether image state resets on route change
  * Whether unnecessary re-renders occur
  * Whether browser caching is bypassed

2. Optimize Profile Photo Loading:

* Ensure:

  * Profile photos load instantly after first fetch
  * Navigation to Profile feels smooth
  * No visible flickering/loading occurs repeatedly
* Prevent:

  * Duplicate image fetches
  * Repeated signed URL generation
  * Unnecessary image refreshes
  * Redundant network requests

3. Implement Smart Caching/Persistence:

* Add/Improve:

  * Client-side image caching
  * Persistent state handling
  * Memoization where appropriate
  * Cached signed URL reuse
  * Optimized image loading strategy
* Ensure:

  * Cache updates automatically after:

    * Upload
    * Delete
    * Replace/update
* Prevent stale image issues.

4. Optimize Component Lifecycle:

* Refactor:

  * Profile component rendering behavior
  * Image state initialization
  * Route transition behavior
* Ensure:

  * Route navigation does not unnecessarily remount/reload the image
  * Profile photo state persists correctly during app navigation
